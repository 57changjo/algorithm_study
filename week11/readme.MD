## 📝 리뷰 요약
- 정리담당: 조민창
### 문제 1: 지름길
- 풀이 접근: DP (Dynamic Programming) 활용. (Dijkstra로도 가능하나, 단방향이고 노드 수가 적어 DP가 직관적)고속도로의 각 위치(0 ~ D)를 노드로 보고, DP[i]를 '위치 i까지 가는 최소 운전 거리'로 정의.
- 주요 코드/아이디어:초기화: DP 배열을 0부터 D까지 idx 값(지름길 없이 가는 거리)으로 초기화.
자료구조: Shortcut 클래스(또는 구조체)를 선언하여 start, end, cost 저장.
도착점(end)이 같은 지름길이 많을 수 있으므로 vector에 push_back으로 관리.
점화식: 현재 위치 i에서,이전 위치에서 그냥 오는 경우: DP[i] = min(DP[i], DP[i-1] + 1)
지름길을 타는 경우(도착지가 i인 지름길 존재 시): DP[i] = min(DP[i], DP[start] + cost)

- 리뷰 포인트: for문 내에서 중복 계산 방지 및 갱신 순서 주의.지름길의 도착 지점이 고속도로 길이 D를 넘는 경우 예외 처리 필수.
Class vs Struct: 코딩 테스트에서는 기본 접근 제어자가 public인 struct가 데이터 묶음 용도로는 더 편리함.

### 문제 2: 공유기 설치
- 풀이 접근: 이분 탐색 (Parametric Search) 활용.
집의 좌표를 탐색하는 것이 아니라, 가장 인접한 두 공유기 사이의 거리를 탐색 범위(low ~ high)로 잡는 것이 핵심.

- 주요 코드/아이디어:결정 함수: 설정한 거리 mid만큼 간격을 띄웠을 때, 목표하는 공유기 개수(C)를 설치할 수 있는가?
가능하면: 거리를 더 벌려봄 (ans = mid, low = mid + 1)
불가능하면: 거리를 좁힘 (high = mid - 1)
- 리뷰 포인트: 이분 탐색 수행 전, 입력받은 집의 좌표는 반드시 정렬(Sort) 되어 있어야 함.
vector::resize 등을 통해 불필요한 메모리 할당을 방지하고 크기를 관리하는 테크닉 필요.

### 문제 3: 키 순서
- 풀이 접근: 모든 정점 쌍 사이의 관계를 파악해야 하므로 플로이드-워셜 (Floyd-Warshall) 알고리즘 사용.
단순히 최단 거리를 구하는 것이 아니라, 노드 간의 도달 가능성(Connectivity)을 확인하는 문제.
- 주요 코드/아이디어:3중 for문 순서: 경유지 노드 k가 반드시 가장 바깥쪽 루프에 위치해야 함 (for k -> for i -> for j).
순위 판별: 나보다 키 작은 사람 수(들어오는 간선) + 나보다 키 큰 사람 수(나가는 간선) == 전체 학생 수 - 1인 경우 자신의 정확한 순위를 알 수 있음.
- 리뷰 포인트:초기화 시 std::fill 함수 활용 권장.
vector::assign(값 덮어쓰기)과 vector::resize(크기 조절)의 차이점 명확히 구분하여 사용.
