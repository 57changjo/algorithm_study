### 문제 1: ATM
Greedy 정렬과 누적합을 통해 전체 대기 시간을 최소화하는 방식으로 문제를 푼다. 각 사람이 돈을 인출하는 데 걸리는 시간을 오름차순으로 정렬하면 대기 시간이 적은 사람들이 먼저 처리되어 뒤의 사람들의 지연 시간이 최소화된다. 정렬된 배열을 기준으로 앞에서부터 누적합을 더하며 각 사람의 총 대기 시간을 계산하고 그 합을 마지막에 출력하면 된다. 벡터 사용을 적극 추천한다 -> sort, push_back 등 벡터 관련 함수들을 사용하여 효율적이고 더 간단하게 문제를 접근 할 수 있다.

### 문제 2: 전자레인지
단순한 나눗셈 연산을 이용한 greedy 접근으로 해결할 수 있다. 버튼 A B C가 각각 300초 60초 10초를 의미하므로 가장 큰 단위부터 가능한 한 많이 사용하는 것이 최소 조작 횟수를 만든다. 풀이 흐름은 간단하다 -> 먼저 T가 10으로 나누어떨어지지 않으면 정확히 맞출 수 없으므로 -1을 출력하고 그렇지 않다면 /와 % 연산으로 순차적으로 A B C 버튼의 횟수와 T를 계산하면서 풀어나간다. 예를 들어 T=1000일 경우 A=3 B=1 C=4처럼 큰 단위부터 나눠가며 남은 시간을 갱신하며 문제를 푼다.

### 문제 3: 톱니바퀴
인접한 톱니의 극성 관계에 따라 회전이 전파되는 과정을 시뮬레이션 문제다. 각 톱니를 문자열 형태로 저장하고 회전 시 인덱스 이동을 처리한다. 회전 명령이 주어질 때 해당 톱니를 기준으로 왼쪽과 오른쪽 방향으로 순차적으로 전파 여부를 판단하고 서로 맞닿은 극이 다를 경우 반대 방향으로 회전 or 같다면 전파 중단. 이때 톱니 배열은 string 벡터로 관리하고 시계/반시계 방향 회전은 rotate() 함수를 사용해 처리한다. 회전 방향 배열을 먼저 계산한 뒤 한 번에 회전시켜야 중간 상태 간섭이 없고 마지막에는 각 톱니의 12시 방향(인덱스 0)을 확인해 1, 2, 4, 8의 가중치를 적용해 총합 점수를 구한다.

### 문제 4: 도영이가 만든 맛있는 음식
완전탐색 + DFS로 풀 수 있다. 시간복잡도는 O(2^N)이지만 N<= 10이어서 완전탐색에 문제가 없다. 문제는 각 재료에 대해 사용한다/사용하지 않는다 두 가지 선택지를 가진다. 모든 조합을 탐색해 신맛(sour *= S[i])과 쓴맛(bitter += B[i])을 누적 계산한다. 최소 차이를 갱신하기 위해 abs(sour - bitter)를 비교하고 아무 재료도 선택하지 않는 경우를 제외해야 한다. 재귀 호출 시 인덱스 증가를 통해 다음 재료로 이동하고 종료 조건에서 최소값을 업데이트한다. climits library를 사용추천 -> INT_MAX와 같은 최대값 상수가 이런 최소값 구하기 문제에 유용하게 사용된다.
